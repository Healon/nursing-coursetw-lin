# AC：來源全選 pill ＋ 已結束課程不再顯示

> 任務日期：2026-07-10。Lin 口頭需求兩點：
> 1. 來源篩選列新增「全選」。
> 2. 已結束課程不再列出（含未來的每次瀏覽都不列）。
> 實作位置：`templates/index.html.tpl`（根目錄 index.html 為 build 產物，隨後重建）。

## AC-1 來源「全選」pill

- [ ] Observable：來源列第一顆 pill 為「全選」，位於所有來源名稱之前。
- [ ] Measurable：點「全選」後，所有來源 pill 的 `aria-pressed="true"`；再點一次全部變回 `false`（回到未篩選）。
- [ ] Bounded：只加在來源組；類別、地區不加。全選＝選取全部來源（顯示筆數與未篩選相同），價值在「先全選、再逐一排除少數來源」的操作路徑。
- [ ] Testable：手動全選所有來源 pill 時「全選」自動亮起；全選後取消任一來源，「全選」自動熄滅；「清除篩選」會一併重置「全選」。

## AC-2 已結束課程不顯示

- [ ] Observable：頁面上不出現任何「已結束」badge 的卡片；今天（daysUntil=0）與未來的活動照常顯示。
- [ ] Measurable：統計列「顯示 X 筆／共 Y 筆」的 Y 只計未結束活動；頁尾各來源筆數 chips 與來源 pill 清單同步以未結束活動計。
- [ ] Bounded：
  - 判定以「訪客開頁當下」為準（client-side），非 build 時間，靜態頁放幾天仍正確。
  - 隨選數位課程（ondemand）無場次日期概念，不受影響、照常顯示。
  - `data/events.json` 的歷史資料**保留不刪**（merge 防清空設計不動），只是不再渲染。
  - 已知邊界：資料模型為單一日期，跨多日活動以起始日判定（與既有「已結束」badge 同一規則，行為一致）。
  - 「已結束」badge 邏輯保留當安全網（開著分頁跨夜、重新篩選時日期重算會用到）。
- [ ] Testable：以含「昨日／今日／明日／隨選」四筆的假資料執行過濾邏輯，結果只剩今日、明日、隨選三筆。

## 驗證方式

1. `pytest tests/` 全綠（既有建置測試不受影響）。
2. `.venv/bin/python scripts/build.py` 成功重建 index.html，內含新 JS。
3. 過濾邏輯以 node（或瀏覽器）實際執行驗證 AC-2 的 Testable 條目。
